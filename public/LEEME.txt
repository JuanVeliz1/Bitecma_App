Resumen de cambios aplicados (BITECMA_MOCKUP_V1.0.3)

1) Arquitectura / “HTML modular”

- Confirmamos y trabajamos sobre el esquema modular: index.html solo monta #root y el layout se arma desde templates/JS (login + app + páginas).
- La navegación y la “BD” se gestionan desde app.js y las páginas desde pages/*.js .
2) Operaciones (OPS) – Transectos y Cuadrantes

- Se reemplazó el flujo de “Agregar transecto/cuadrante” por interfaces más rápidas:
  - Transectos: modal con tabla tipo “botes” para crear múltiples, luego se evolucionó a:
    - Sin checkbox “Usar”.
    - Botón “Agregar transecto” (de a uno) y botón “Eliminar” por fila.
    - Área en combo: 120/60/50/30/20 + “Otro…” (input manual).
    - Selector múltiple de especies por fila y creación con especies pre‑cargadas (counts inicial en 0).
  - Cuadrantes: se cambió a creación masiva desde una tabla de 1 fila (cantidad + área + sustrato + especie) para generar N cuadrantes rápidamente.
3) Coordenadas y fecha

- Se eliminó la captura/uso de coordenadas (X/Y/Lat/Lon) en el flujo de unidades.
- La fecha de creación se dejó automática (fecha del sistema), sin pedirla en formularios.
4) Portadas optimizadas (sin “abrir” unidad)

- Cuadrantes:
  - Se rediseñó la “portada” para que todos los datos relevantes estén a la vista.
  - Input numérico vacío para “Cantidad” por especie (si 0), con navegación por teclado:
    - ENTER guarda y salta al siguiente cuadrante.
  - Combo de área del cuadrante + combo de sustrato en portada.
  - Se eliminó cubierta biológica para cuadrantes.
  - Se eliminó abrir/cerrar detalle y también botones de editar/cambiar especie en cuadrante.
  - Se ajustó el texto: nombre de especie se muestra en portada (reemplazando la fecha) y “Cantidad:” etiqueta el input.
- Transectos:
  - Portada con combos editables: Área (m²), Sustrato, Cubierta biológica.
  - Dentro del transecto quedó solo “Conteo de especies” (el resto se eliminó del cuerpo).
5) Botes

- En “Crear botes” se aumentó la lista predeterminada inicial (se agregó 1 bote adicional).
- Se ordenó el render de botes por zona dentro de cada operación.
- Se ajustó el encabezado del bote al orden: Nombre bote · Nombre buzo · Zona .
- Se corrigió el tab inicial del bote para que si hay muestras L‑P, abra por defecto L‑P (en vez de densidad).
6) Eliminación con confirmación

- Se agregó botón para Eliminar bote desde “Editar bote” con confirmación.
- Se agregó botón Eliminar al final de cada cuadrante y de cada transecto , con confirmación antes de borrar.
7) Datos (operaciones/EVADIR)

- Se eliminaron ejemplos hardcodeados y se reemplazaron por operaciones derivadas de los Excel, inicialmente como data en app.js .
- Luego se movió la data a un archivo externo JS:
  - Se consolidó el uso de data/db.js como “base de datos” cargada vía <script> (para funcionar sin servidor local).
  - app.js consume las operaciones desde window.DB_DATA (o similar según el estado actual del archivo).
- La página EVADIR dejó de usar filas de ejemplo y se ajustó para renderizar en base a las operaciones disponibles.
- Se actualizaron conteos/estados visibles en dashboard para reflejar el dataset real (y luego se fue ajustando según operaciones activas).
8) Ajustes específicos de datos

- Se agregaron manualmente datos faltantes de peso‑longitud/diámetro (Rincón de Ollas) y luego, por decisión, se eliminó la operación “Rincón de Ollas” del mockup dejando solo 2 operaciones.
- Para “MAR Y CIELO – BAHIA CHINCUI” se ajustó sustrato en cuadrantes a ROCA/ARENA (cuando se trabajó con el set reducido).

9) Consumo de catálogos desde data/ (sin hardcode en app.js )
  - index.html ahora carga data/sectores.js y data/especies.js después de data/db.js y antes de components/app.js : index.html
  - components/app.js pasa a leer:
    - ESPECIES desde window.DB.especies (de data/especies.js )
    - REGIONES_CHILE desde window.DB.regionesChile (de data/sectores.js )
    - caletas base desde window.DB.caletasByRegionStatic (de data/sectores.js )
  - DB de operaciones se toma desde window.DB.operaciones (compatibilidad con data/db.js ): app.js
- Eliminación de duplicación de caletas
  - Se vació el dataset hardcodeado CALETAS_BY_REGION_STATIC dentro de app.js para que la fuente sea data/sectores.js : app.js
Hoy (documentación + mejoras EVADIR en L‑P/Longitud)

10) Documentación por código en components/
  - Se agregaron descripciones breves arriba de cada función (estilo JSDoc) en:
    - svgIcon.js , toast.js , modal.js , topbar.js , sidebar.js , contenedorPaginas.js , templates.js
  - En app.js se documentaron helpers y window.* por secciones: app.js

11)EVADIR: separación real de “solo Longitud” vs “Peso‑Longitud”
  - En la previsualización EVADIR se separan las tablas en dos secciones:
    - Tablas Longitud
    - Tablas PESO‑LONGITUD
  - Si una misma especie tiene registros mezclados (algunos con peso y otros sin peso), ahora se dividen en tablas distintas al generar lpSections : app.js
  - En las tablas de solo longitud se elimina completamente la columna PESO (header + celdas): app.js
  - Si no hay registros de solo Longitud , ya no se muestra nada de esa sección (ni título, ni botón Exportar, ni infobox): app.js
